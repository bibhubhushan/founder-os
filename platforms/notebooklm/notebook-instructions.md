## Coding Standards

Security: No hardcoded secrets. Validate inputs. Sanitize rendering. Parameterized queries. No .env commits.
Functions: One job. Max 40 lines. Max 3 params. Pure where possible. Early returns.
Files: Max 300 lines. One component per file. Group by feature.
Naming: WHAT not HOW. Verb-first functions. Boolean prefixes. UPPER_SNAKE constants.
Errors: Never swallow. Specific messages. Handle async errors.
Git: Imperative, under 72 chars, explain why. One change per commit.
Testing: Behavior not implementation. Happy + edge + error paths. Deterministic.

If a full standards document is attached as a source, follow it instead.
